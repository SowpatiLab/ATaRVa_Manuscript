# TAMATR - Tool for Aggregating & Merging ATaRVa TR_vcfs

<!-- <p align=center>
  <img src="logo_tamatr.png" alt="Logo of Mirchi" width="200"/>
</p> -->

Tamatr is a tool for merging VCF files generated by ATaRVa. It is built using the `pysam` & `polars` packages and supports multi-threaded execution for improved performance.

The tool requires the following inputs:

- BGZipped and tabix-indexed VCF files

- A BGZipped and tabix-indexed BED file specifying the regions of interest

## Usage
```bash
python tamatr.py -h
```
which gives the following output:

```
usage: hemi_tamatr.py [-h] -r <FILE> -i <FILE> [<FILE> ...] -f <FILE> [--contigs CONTIGS [CONTIGS ...]] [-o <STR>] [-p <INT>]

Required arguments:
  -r <FILE>, --regions <FILE>
                        input regions file. the regions file should be strictly in bgzipped tabix format. If the regions input file is in bed format. First sort it using bedtools. Compress it using
                        bgzip. Index the bgzipped file with tabix command from samtools package.
  -i <FILE> [<FILE> ...], --vcfs <FILE> [<FILE> ...]
                        text file containing paths to input vcf files to be merged. The text file should list each path on a separate line. The vcf files should be strictly in bgzipped tabix format. If
                        the vcfs input file is in vcf format. First sort it using bcftools. Compress it using bgzip. Index the bgzipped file with tabix command from samtools package.
  -f <FILE>, --fasta <FILE>
                        input reference fasta file. The file should be indexed.

Optional arguments:
  --contigs CONTIGS [CONTIGS ...]
                        contigs to get merged [chr1 chr12 chr22 ..]. If not mentioned every contigs in the region file will be merged.
  -o <STR>, --outname <STR>
                        name of the output file, output is in vcf format.
  -p <INT>, --processor <INT>
                        number of processor. [default: 1]
```
**NOTE: TAMATR will merge only those loci that are present in the input BED file**

## Examples
### Basic usage
```bash
python tamatr.py -i file1.vcf.gz file2.vcf.gz file3.vcf.gz -r TR_regions.bed.gz -f reference.fasta
```

### With vcf text file
```bash
python tamatr.py -i vcf_files.txt -r TR_regions.bed.gz -f reference.fasta
```

### With multithreading
```bash
python tamatr.py -i file1.vcf.gz file2.vcf.gz file3.vcf.gz -r TR_regions.bed.gz -f reference.fasta -p 4
```
In all the above examples, the output of Tamatr is saved to atarva_merged.vcf unless `-o or --outname` is specified.
